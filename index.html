Resumen ejecutivo

El objetivo es lanzar un sitio que represente la marca de forma premium y convierta visitas en acción: descubrir recetas, conocer productores, explorar productos, y ubicar bares donde se consume el mezcal.

Resultado esperado: un sitio rápido, responsivo y fácil de mantener, con contenido editable por el cliente, y una experiencia fluida con animaciones sutiles.

Alcance (incluido)

  • Diseño moderno, limpio y artesanal (alineado al brand deck o referencias visuales).
  • Home con storytelling + CTA (productos / virtual bar / mapa / newsletter).
  • Productos con fichas técnicas por “expresión” (galería, notas, specs).
  • Virtual Bar: recetas, historias y experiencias (con estructura escalable).
  • Productores en Oaxaca: sección editorial + contenido multimedia.
  • Mapa de bares (Google Maps) + listado de bares.
  • Newsletter / captación: formularios (integración a plataforma de email si aplica).
  • Fotos y videos integrados (óptimos para performance).
  • Animaciones básicas + microinteracciones (sin perjudicar velocidad).
  • SEO básico (metas, estructura, performance) + setup técnico (dominio/publicación).

*El alcance exacto queda cerrado en el Statement of Work (SOW) al iniciar.

Opciones de implementación

Opción A — Webflow (recomendado)

Ideal si el equipo quiere autogestionar contenido (CMS/Editor) sin depender de desarrollo.

Entrega más rápida Edición simple Menos mantenimiento técnico Excelente para marketing
  • Contenido editable: Productos / Recetas / Productores / Bares.
  • Mapa de bares: Google Maps + datos desde CMS.
  • Actualización desde planilla: Google Sheets → (Make/Zapier) → Webflow CMS (opcional).

Opción B — 100% Programado (Full Code)

Ideal si buscan máxima flexibilidad y escalar features e integraciones a futuro.

Flexibilidad total Escalabilidad Performance top Integraciones avanzadas
  • Arquitectura moderna (ej. Next.js) + CMS (Sanity/Strapi) o panel admin.
  • Mapa de bares: Google Maps + fuente de datos (CMS o Sheets).
  • Requiere mayor QA, deploy y mantenimiento técnico.

Actualización de contenido y Google Sheets

Webflow (CMS/Editor)

El cliente edita contenido desde el Editor/CMS (sin tocar diseño). Para bares: se puede automatizar desde Google Sheets usando Make/Zapier para crear/actualizar items en la colección “Bares”.

Full Code (CMS o Panel)

El cliente edita desde un CMS (Sanity/Strapi) o panel admin. La planilla (Sheets) puede usarse como fuente, pero lo más estable es centralizar contenido en CMS y usar Sheets solo como input si se requiere.

*Servicios externos (Make/Zapier, email marketing, etc.) pueden tener costo mensual según volumen.
